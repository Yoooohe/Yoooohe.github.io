    How to establish github page?

    reference page

    Note-yuhe:
    [] in command indicates that content in brackets can be changed according to your choice. These brackets are not actually existing in command.

    1. Install ++Homebrew++

    homebrew is a package management tool with Ruby underneath.

    (1) One simple way to install it:
    $/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

    (2) To check wether we have installed it successfully:
    $brew --version or brew -v
    If successfully:

    (3) More brew command:

    2. Install ++Git++

    In mac, there is already one version of git. But we can also install other git with brew. The installed git will have a higher priority than the local one in mac.

    (1) Install Git with brew:
    $ brew install git

    (2) Check its installment status:
    $ git --version

    3. Install certain version of ++node++ with ++nvm++

    nvm is node managment tool in mac, very similar to rvm of Ruby.
    (1) install nvm:
    $ brew install nvm

    (2) configure nvm:
    edit ~/.bash_profile, if not existed, create one.
    $ cd ~, vim .bash_profile
    (note-yuhe: I usually use nano to edit file)
    In .bash_profile, we need to add commands below:

    $ nvm ls-remote #check all available version
    $ nvm install [v8.4.0] #the version you want
    $ nvm use [v8.4.0] #use certaion version of node
    $ nvm alias default [v8.4.0] #default version when open termial

    $ cd ~
    $ mkdir github-workspace
    $ mkdir yoooohe.github.io

    files:
    _config.yml, db.json, package-lock.json, package.json

    directories:
    node_modules. public, scaffolds, source, themes

    $ cd themes
    $ git clone [git@github.com:forsigner/fexo.git themes/fexo]

    ##edit _config.yml, change theme: [fexo]

    ##update theme
    $ cd themes/[fexo]
    $ git pull

    ##change avatar
    theme/[fexo]/source/images/avatar.jpg

    ##republish hexo to github page
    $ hexo clean
    $ hexo g #hexo generate
    $ hexo d #hexo deploy

    Note-yuhe: in order to make hexo d function, we need to

    npm install hexo-deployer-git

    $ hexo g
    $ hexo s

    $ hexo server //start the server
    $ hexo s #simplified version of previous one
    $ hexo server -s #static mode
    $ hexo server -p 5000 #change port
    $ hexo server -i 192.168.1.1 # customize IP address

    ##INFO Start processing

    ##INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

    $ hexo clean #clean cache
    $ hexo new page [“file”] #create new directory called file and automatically create index.md under that directory

    ##create new blog
    $ hexo n fileName
    $ hexo new [“fileName”]
    $ hexo new post fileName
    $ hexo new post [“fileName”]

    ##generate/compile/publish (generate static web page)
    $ hexo generate
    $ hexo g
    $ hexo generate watch
    $ hexo deploy
    $ hexo d

    ##deploy after generate

    ##two commands have same effect
    $ hexo generate –deploy
    $ hexo deploy –generate
    $ hexo deploy -g
    $ hexo server -g

    $ npm install hexo-deployer-git –save
    $ git init
    $ git remote add origin https://github.com/Yoooohe/Yoooohe.github.io.git

    when first use git, git may require us to input username and password, username is our registered email.

    $ hexo clean && hexo g && hexo d
    $ git add
    $ git commit -m [‘content’]

    _config.yml:

    Deployment

    Docs: http://hexo.io/docs/deployment.html

    deploy:
    type: git
    branch: master
    repo: https://github.com/Yoooohe/Yoooohe.github.io.git

    Finally, my website~
    https://yoooohe.github.io
